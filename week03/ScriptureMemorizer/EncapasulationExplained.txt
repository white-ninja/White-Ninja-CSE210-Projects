Encapsulation is an object-oriented programming principle that involves bundling data (variables)
and the methods that operate on that data into a single unit,
typically a class, while restricting direct access to the internal state of the object.
This is achieved using access modifiers such as `private` and `public`. 
By making fields private and exposing controlled methods to interact with them,
encapsulation protects the integrity of the data and ensures that it is only modified in intended ways.

One major benefit of encapsulation is data protection and control. 
It prevents other parts of the program from directly altering an object’s internal state, 
which reduces bugs and makes the program easier to maintain. 
It also allows developers to change the internal implementation of a class without affecting other parts of the program, improving flexibility and scalability.

In my Scripture Memorizer program, encapsulation is demonstrated in the `Word` class.
The internal state of a word is hidden, and controlled methods are provided to interact with it:

public class Word
{
    private string _text;
    private bool _isHidden;

    public Word(string text)
    {
        _text = text;
        _isHidden = false;
    }

    public void Hide()
    {
        _isHidden = true;
    }

    public bool IsHidden()
    {
        return _isHidden;
    }
}

In this example, the fields _text and _isHidden are private, meaning they cannot be accessed directly from outside the class.
Instead, methods like `Hide()` and `IsHidden()` provide controlled ways to modify and view the state of the object. 
This ensures that the logic for hiding a word is managed within the class itself, maintaining consistency and preventing unintended misuse. 
This is a practical application of encapsulation, as it keeps the data secure while still allowing necessary interaction through well-defined methods.
